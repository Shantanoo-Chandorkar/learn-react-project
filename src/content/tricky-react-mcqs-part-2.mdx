### Q7 — JavaScript in Head (async vs defer)

**Question:**
Two scripts are in `<head>`: `file1.js` and `file2.js`. `file2.js` depends on `file1.js`. Loading them is slowing down rendering. The HTML can render without JS (fallbacks exist). Which statements are true? (Pick all that apply)
1. It is not possible for the webpage to start rendering before loading the JS files, as long as `<script>` is in `<head>`
2. This problem can be fixed by using `async` on both `<script>` tags
3. This problem can be fixed by using `defer` on both `<script>` tags
4. This problem can be fixed by using `async` for file1.js and `defer` for file2.js
5. This problem can be fixed by using `defer` for file1.js and `async` for file2.js

**Answer: Option 3 only**

- `async`: parallel download, executes immediately when ready, order NOT guaranteed
- `defer`: parallel download, executes after HTML parsing, order preserved
- ❌ Option 1: Both async and defer allow rendering to proceed
- ❌ Option 2: `async` on both doesn't guarantee order
- ✅ Option 3: `defer` on both — no render blocking + order preserved
- ❌ Option 4: `async` on file1 could execute after file2 (defer)
- ❌ Option 5: `async` on file2 could fire before file1 resolves

### Q15 — Promises with async/await

**Question:**
```javascript
var async_f = async function() {
  var p1 = promised_function('first');
  var p2 = promised_function('second');
  console.log(await p1);
  console.log(await p2);
  console.log('done')
}
async_f();
```
Where `promised_function` resolves after 3 seconds. What happens? (Select closest option)
1. After 3 seconds, first, second, and done are all printed together
2. done is printed immediately. After 3 seconds first is printed. After 3 more seconds second is printed.
3. done is printed immediately. After 3 seconds, first and second are both printed together.
4. After 3 seconds, first is printed. After 3 more seconds, second and done are printed together.
5. done is printed immediately. 6 seconds later, first and second are both printed together.

**Answer: Option 1**

- Both `p1` and `p2` timers start concurrently before any `await`
- `await p1` pauses ~3 seconds → prints `first`
- `await p2` is already resolved by then → prints `second` almost instantly
- `console.log('done')` fires right after
- All three print at the ~3 second mark

### Q16 — Angular URL #1

**Question:**
An Angular app is accessed as `https://example.com/myapp?query=string`. The component has `constructor(private rt: ActivatedRoute) {}`. Which is the right way to get the `query` parameter value in `ngOnInit`? (Select closest option)
1. `const qStr: string = this.rt.snapshot.params.query`
2. `const qStr: string = this.rt.snapshot.params.get('query')`
3. `const qStr: string = this.route.snapshot.queryParamMap.get('query')`
4. `const qStr: string = this.rt.snapshot.queryParamMap.get('query')`
5. `const qStr: string = this.rt.snapshot.queryParamMap.get('query=string')`

**Answer: Option 4**

- `?query=string` is a query parameter, not a route param
- `snapshot.params` is for route params like `/path/:id`
- `queryParamMap.get()` is the correct API for query params
- `this.route` doesn't exist — injected reference is `rt`
- Option 5 is wrong — pass only the key name, not `'query=string'`

### Q20 — Pipe Parameter

**Question:**
```typescript
@Pipe({name: 'discount'})

  transform(value: cost, percent?: number): number {
```

### Q26 — Expensive Service

**Question:**
Given a JSON string like:
```json
[
  ["Swiggy", 123], ["Swiggy", 227], ["Zomato", 103],
  ["Zomato", 171], ["Dunzo", 131], ["Zomato", 122], ["Swiggy", 181]
]
```
Write a JavaScript function that takes such a string and returns the name of the service with the highest average payment. For the above input, it should return "Swiggy". Aim for O(n) time complexity.

**Answer:**

```javascript
// Time complexity: O(n)
function mostExpensiveService(jsonString) {
  const payments = JSON.parse(jsonString);
  const map = new Map();

  for (let i = 0; i < payments.length; i++) {
    const service = payments[i][0];
    const amount = payments[i][1];
    if (!map.has(service)) {
      map.set(service, { total: 0, count: 0 });
    }
    map.get(service).total += amount;
    map.get(service).count += 1;
  }

  let bestService = null;
  let bestAverage = Number.NEGATIVE_INFINITY; // ensures any real average will be higher on first comparison

  for (const [service, data] of map) {
    const avg = data.total / data.count;
    if (avg > bestAverage) {
      bestAverage = avg;
      bestService = service;
    }
  }

  return bestService;
}
```

Verification:
- Swiggy: (123 + 227 + 181) / 3 = 177
- Zomato: (103 + 171 + 122) / 3 = 132
- Dunzo: 131 / 1 = 131

Returns "Swiggy" ✓
